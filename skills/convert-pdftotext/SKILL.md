# PDF 轉 Markdown

將使用者提供的 PDF 檔案讀取並轉換為乾淨、結構化的 Markdown（.md）文件。

## 使用方式

```
/convert-pdftotext <pdf_file_path> [output_path]
```

- `pdf_file_path` — PDF 檔案的絕對路徑（必填）
- `output_path` — 輸出 .md 檔案路徑（選填，預設為同目錄同檔名 .md）

## 轉換流程

1. **讀取 PDF** — 使用 Read 工具讀取 PDF 內容（大型 PDF 分頁讀取，每次最多 20 頁）
2. **解析結構** — 辨識以下元素：
   - 標題層級（H1–H4）
   - 段落文字
   - 表格（轉為 Markdown 表格語法）
   - 項目清單（有序/無序）
   - 粗體、斜體等行內格式
3. **輸出 Markdown** — 將解析結果寫入 .md 檔案

## 轉換規則

| PDF 元素 | Markdown 對應 |
|----------|--------------|
| 大標題 / 封面標題 | `# 標題` |
| 章節標題 | `## 標題` |
| 子章節標題 | `### 標題` |
| 段落文字 | 一般段落（空行分隔） |
| 表格 | Markdown 表格語法 `\| col \|` |
| 項目符號清單 | `- 項目` |
| 編號清單 | `1. 項目` |
| 頁首/頁尾/頁碼 | 移除 |

## 大型 PDF 處理

- PDF 超過 10 頁時，分批讀取（每批 20 頁）
- 逐批解析並合併輸出
- 確保跨頁內容的連續性

## 輸出格式

輸出的 Markdown 檔案包含：

```markdown
# [文件標題]

> 來源：[原始 PDF 檔名]
> 轉換日期：[YYYY-MM-DD]
> 頁數：[總頁數]

---

[轉換後的內容]
```

## 注意事項

- 掃描式 PDF（圖片型）無法直接提取文字，會提示使用者
- 複雜排版（多欄、浮動圖片）可能需要手動調整
- 圖片不會被提取，僅保留圖片位置標記 `[圖片：描述]`
